## How it fits together

The plugin runs inside your OpenClaw gateway and hosts a local WebSocket server. A hosted service -
**the StandIn media bridge** - joins the Teams call and dials into that server for each call. You
never expose Teams internals yourself; you run the plugin, point StandIn at it with a shared secret,
and the agent is in the call.

```
Teams call <-> StandIn media bridge ==WebSocket==> @komaa/msteams-voice <-> your OpenClaw agent
```

## The three pillars

<CardGrid>
  <Card title="Perception - it sees you" icon="magnifier">
    Reads inbound camera and screen-share (VBSS) frames - on demand, per-turn, or as a continuous
    ambient view - attributed per participant and capped per call.
  </Card>
  <Card title="Dialogue - it talks with you" icon="comment">
    Natural spoken conversation, realtime speech-to-speech or streaming STT to agent to TTS, with
    barge-in, a "speak only when addressed" gate for meetings, DTMF, and bilingual EN/AR.
  </Card>
  <Card title="Rendering - it is seen by you" icon="seti:video">
    Appears as a lip-synced avatar tile; emits expression cues, visemes, and image overlays that the
    StandIn bridge draws into the caller's video.
  </Card>
</CardGrid>

## Capability highlights

- **Two dialogue modes** - realtime (OpenAI / Azure OpenAI) or streaming with any provider.
- **Vision** - camera + screen-share, ambient view, keyframe history, per-call budget.
- **Meeting etiquette** - wake-phrase gate, follow-up window, per-speaker attribution.
- **Outbound call-backs** - place a call, speak a result, hang up, with voicemail fallback.
- **Meeting recap + `.docx` minutes** with per-person attribution.
- **Avatar cues** - expressions, viseme lip-sync, picture-in-picture image sharing.
- **Secure transport** - replay-proof HMAC handshake, allowlist closed by default, recording-status gate.
